# Bilim OS — Проактивная мультиагентная система: панель агентов по ролям

> Дата: 2026-06-01. Статус: **дизайн / ТЗ** (код по фазам ниже).
> Заземлено на реальные модули прода (оценки, посещаемость, КТП, поурочные планы, тесты, заявления, опросы, инциденты, отчёты).
> Это HEADLINE для КП: «школа как живой организм» — событие → действие, у каждой роли свой ИИ-копилот, человек подтверждает.

---

## 1. Идея в одном абзаце

Платформа не ждёт, пока человек что-то спросит. Любое **событие** в системе (учитель поставил «2», ученик пропустил 3 дня, тест провален, КТП отстаёт) попадает в **шину событий**, **движок правил** решает, что с этим делать, и **исполнитель действий** создаёт для нужной роли элемент в её **Панели агента**: тихий алерт, задачу, готовый черновик письма или рекомендацию. Тяжёлое (эмпатичный текст родителю, поиск пробела в знаниях) дописывает LLM, но **человек подтверждает** всё, что уходит наружу. Без LLM-ключа всё работает на шаблонах.

```
СОБЫТИЕ ──► AgentEvent (лог) ──► Движок правил (AgentRule) ──► Действие ──► AgentItem в Панели роли ──► человек: ✓ подтвердить / ✗ отклонить ──► AgentActionLog (аудит)
                                                                  └─ (опц.) LLM-обогащение черновика
```

## 2. Архитектура (бэкенд)

### 2.1 Модели Prisma (предлагаемые)

```prisma
model AgentEvent {
  id          String   @id @default(cuid())
  type        String   // grade.created | attendance.absent | test.failed | ktp.behind | ...
  actorUserId String?  // кто инициировал (учитель и т.п.)
  studentId   String?  // к кому относится
  classId     String?
  payload     Json     // {value, subjectId, testId, ...}
  createdAt   DateTime @default(now())
  processedAt DateTime?
  @@index([type, processedAt])
}

model AgentRule {
  id          String   @id @default(cuid())
  key         String   @unique         // low-grade-parent-alert
  title       String
  eventType   String                   // на какое событие реагирует
  condition   Json                     // {field:"value", op:"lte", value:2}
  actions     Json                     // [{kind:"alert", forRole:"parent"}, {kind:"task", forRole:"teacher"}]
  enabled     Boolean  @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model AgentItem {        // элемент Панели агента (инбокс роли)
  id          String   @id @default(cuid())
  ruleKey     String?
  eventId     String?
  forUserId   String?  // конкретный получатель
  forRole     String?  // или роль (если адресат вычисляется позже)
  studentId   String?
  kind        String   // alert | task | suggestion | draft
  severity    String   @default("info") // info | warn | urgent
  title       String
  body        String   @db.VarChar(2000)
  payload     Json?    // черновик письма, ссылка на тест и т.п.
  status      String   @default("new")  // new | in_progress | approved | done | dismissed
  createdAt   DateTime @default(now())
  resolvedAt  DateTime?
  resolvedBy  String?
  @@index([forUserId, status])
  @@index([forRole, status])
  @@index([studentId])
}

model AgentActionLog {   // аудит «что агент сделал»
  id        String   @id @default(cuid())
  itemId    String?
  action    String   // created | approved | sent | dismissed | escalated
  byUserId  String?  // null = system
  detail    Json?
  createdAt DateTime @default(now())
}
```

### 2.2 Поток (3 слоя)

1. **Эмиттеры событий** — тонкие вызовы `emitEvent(type, ctx)` в существующих API (POST /grading → `grade.created`; POST /attendance → `attendance.marked`; tests submit → `test.completed/failed`). Пишут `AgentEvent`.
2. **Процессор** — `processEvents()` берёт необработанные события, матчит активные `AgentRule`, исполняет действия → создаёт `AgentItem`. Запуск: синхронно после эмита (MVP) или по расписанию/очереди (позже).
3. **Действия (Action catalog)** — `alert` (тихий алерт в панель), `task` (задача с дедлайном), `suggestion` (рекомендация: сгенерить ремедиал-тест, найти пробел), `draft` (черновик письма родителю — опц. LLM). Все адресные действия уважают **teacher-scope/RBAC** (агент не видит больше, чем роль).

### 2.3 Человек в контуре + безопасность
- Всё **исходящее наружу** (письма/пуши родителю) — только через подтверждение. Агент готовит → человек жмёт «Отправить».
- Полный **аудит** действий агента (`AgentActionLog`).
- Приватность: агент работает в границах роли; чувствительное (буллинг/психология) — только тихий алерт специалисту, без раскрытия деталей другим.
- LLM-слой опционален (OpenRouter; стаб-шаблоны без ключа). Перспектива из манифеста: два контура — локальный (Llama, конфид. данные) + облачный (тяжёлая генерация).

## 3. Панель агента по ролям

Единый компонент «Панель агента» (инбокс `AgentItem` для текущей роли) + специфичные триггеры. Ниже — **на какие события реагирует** и **что делает** агент каждой роли.

### 👦 Ученик — ИИ-тьютор
| Триггер | Действие агента |
|---|---|
| Провален тест / низкий балл | «Разберём тему за 5 минут?» → объяснение + 3 тренировочных вопроса (НАВОДИТ, не даёт готовый ответ) |
| Незакрытое ДЗ к сроку | Напоминание + подсказка с чего начать |
| Системно слабая тема (по истории оценок) | «Дефрагментация знаний»: спуск по графу к базовому пробелу, мини-план подтяжки |
| Зашёл в тьютор | Сократический диалог, генерация практики из слабых мест |
- **Данные:** свои оценки, тесты, ДЗ, расписание. **LLM:** да (с ключом — реальный диалог, без — шаблонные подсказки/готовые тесты).

### 👩‍👦 Родитель — агент-информатор
| Триггер | Действие |
|---|---|
| Ребёнку поставили «2» | Уведомление + спокойный контекст (по какому предмету, что это значит) + «что сделать дома» |
| Непокрытый пропуск | Алерт + кнопка «отправить записку/заявление» |
| Рост/достижение | Позитивное сообщение (не только негатив!) |
| Раз в неделю | Дайджест: оценки, посещаемость, настроение, рекомендации |
- Сюда же приходят **подтверждённые учителем** письма. **LLM:** эмпатичный текст (стаб = шаблон).

### 👨‍🏫 Учитель — агент-помощник (частично уже в проде)
| Триггер | Действие |
|---|---|
| Поставил «2» / серию низких | Готовит **черновик письма родителю** (на подтверждение) + предлагает ремедиал-тест |
| Средний балл класса просел | Флаг + срез: кто тянет вниз |
| Завтра уроки без темы/плана | Предлагает сгенерировать **поурочный план/презентацию** (готово: `/lesson-plans`, `/presentations`) |
| КТП отстаёт от расписания | Напоминание + предложить сдвиг тем |
| (Перспектива) Фото тетради | OCR → черновая проверка |
- **Данные:** свои классы/предметы (teacher-scope). **LLM:** презентации, поурочные планы, письма, отчёты родителям.

### 🧑‍🏫 Куратор — агент классного руководителя
| Триггер | Действие |
|---|---|
| Несколько пропусков/инцидентов в классе | Сводка «кому нужно внимание на этой неделе» |
| Маркеры буллинга (падение посещаемости + оценок + настроение) | Тихий алерт + предложить связаться с психологом |
| Висят несобранные согласия/заявления | Список + черновик рассылки родителям класса |

### 🧑‍💼 Завуч — агент контроля и оркестрации
| Триггер | Действие |
|---|---|
| Конфликты нагрузки/дыры в расписании | Алерт + вариант замены |
| КТП отстаёт у нескольких учителей | Сводный статус |
| Аномалии оценок (накрутка/занижение) | Флаг на модерацию |
| Школьный провал посещаемости | Алерт + разрез по классам |
| Эскалация снизу | Принять решение / передать |
- **Данные:** вся школа. Генерация отчётов в МинОбр/Кундолюк (из манифеста — авто-выгрузка).

### 🧠 Психолог/специалист — «Предиктивный радар»
| Триггер | Действие |
|---|---|
| Косвенные маркеры стресса (посещаемость −15%, падение оценок, маркеры в голосе тьютора) | **Тихий** упреждающий алерт ДО открытого конфликта |
| Анонимный скрининг (текстовый шлюз) | Заявка в работу |
| Открыт кейс | Тайм-лайн настроения, рекомендации вмешательств |
- **Приватность максимальная:** детали видит только специалист.

### 🗂️ Секретарь — агент делопроизводства
| Триггер | Действие |
|---|---|
| Неполная карточка ученика / нет документов | Список пробелов + что запросить |
| Новые заявления/заказы столовой | Маршрутизация + черновики ответов |
| Нужна справка/документ | Авто-генерация по шаблону |

### 📊 Аналитик — агент инсайтов
| Триггер | Действие |
|---|---|
| Аномалия в метриках школы | Авто-отчёт с трендом |
| Запрос среза | Ad-hoc аналитика, предиктивные дашборды |
| Сбор data-driven case (B2G) | Подготовка доказательной базы для питча (из roadmap манифеста) |

### 🛡️ Суперадмин — агент системы и правил
| Триггер | Действие |
|---|---|
| Аномалии доступа/здоровья системы | Алерт |
| — | Управление **каталогом правил агента** (`AgentRule`), аудит действий, глобальный конфиг, вкл/выкл сценариев |

## 4. Эскалационные цепочки (примеры «живой системы»)
- **Успеваемость:** ученик провалил тест → тьютор предлагает разбор → если не закрыто N дней → задача учителю → если паттерн → алерт куратору → завуч.
- **Благополучие:** пропуски↑ + оценки↓ → радар психолога (тихо) → психолог работает → фокус восстановлен → оценка растёт (замкнутый цикл из манифеста).
- **Низкая оценка:** «2» → черновик письма родителю (учитель ✓) → родителю + рекомендация ученику.

## 5. План реализации по фазам

- **Фаза A — Фундамент (без LLM):** модели `AgentEvent/AgentRule/AgentItem/AgentActionLog` + миграция; `emitEvent()` в POST /grading и /attendance; `processEvents()` синхронно.
- **Фаза B — Первые правила:** 3 сид-правила — `grade ≤ 2 → alert родителю + задача учителю`; `≥3 пропусков подряд → alert куратору`; `тест провален → suggestion ученику (ремедиал)`.
- **Фаза C — Панель агента (UI):** компонент инбокса + бейдж-счётчик в шапке (есть IconBell); экран по роли со статусами new/approve/dismiss; раздел «Согласовать письмо родителю».
- **Фаза D — LLM-обогащение:** эмпатичные черновики, «дефрагментация знаний», тьютор — через OpenRouter (стаб без ключа).
- **Фаза E — Радар и эскалации:** предиктивный радар психолога, цепочки эскалации, недельные дайджесты, авто-выгрузка отчётов.
- **Фаза F — Каталог правил:** UI суперадмина для правил + аудит.

**MVP для демо/КП = A+B+C** (работает без ключа, видно «магию» событие→действие). D+ добавляют реальный ИИ.

## 6. Тезисы для КП
- **Проактивность:** система сама инициирует помощь, а не ждёт запроса.
- **Мультиагентность:** у каждой роли — свой копилот, все связаны в один поток данных.
- **Человек решает:** агент готовит, человек подтверждает (доверие + безопасность).
- **Работает сразу:** базовые сценарии без облака; ИИ-генерация подключается ключом.
- **Дипломатия с государством:** сухая статистика авто-уходит в Кундолюк, инновации остаются у нас (из манифеста).
